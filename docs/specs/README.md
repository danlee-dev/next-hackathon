# docs/specs/

기능별 INTENT 문서. 새 기능 시작 시 `/sketch <feature>` 또는 직접 작성.

## 형식

```markdown
# <feature-slug>

## Intent
한 줄로 *무엇을 왜 만드는지*.

## Scope
- in: 이 기능이 다루는 것
- out: 명시적으로 빼는 것

## Acceptance
- [ ] 사용자가 X 할 수 있다
- [ ] Y 검증된다

## Notes
참고 / 결정 / 트레이드오프
```

## 기존 문서

(아직 없음 — 첫 spec은 `/sketch <slug>` 로)
